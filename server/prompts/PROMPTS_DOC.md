# Dungeon Master Prompt Catalog

This document describes server-side prompt templates, layout, and where they load. Edit files under `server/prompts/`; the server prefers these over any client copies.

## Directory layout

| Folder | Role |
|--------|------|
| `core/` | Always-on DM persona and global style (`system`, `style`). |
| `rules/` | Guards, language directives, memory wrapper, intent router, scene grounding. |
| `skills/` | Mode-specific skills (combat, investigation, decision, adventure seed, character gen). |
| `templates/dm/` | Mustache injections, response envelope, party/PC context, combat handoff. |
| `templates/campaign/` | Campaign core generator (merged file + stage templates). |

## Loading

- `promptManager.loadPrompt(name)` expects a path relative to `server/prompts/` (e.g. `rules/language_spanish.txt`, `templates/dm/inject_initial.txt`).
- `loadCampaignGeneratorParts()` reads `templates/campaign/generator.txt`, splits on a single `---` line: **build context** (system, world-only JSON rules) and **user template** (Mustache with `{{sessionSummary}}`, `{{languageInstruction}}`). `/generate-campaign` and `/generate-campaign-core` use both slices (build + full user text as system priming, then rendered user text as the user message), matching the old two-file behavior.

## Flow notes

- System messages are consolidated via `consolidateSystemMessages(...)` before sending to the model to preserve guard precedence.
- Campaign core is generated by `/generate-campaign-core` using `templates/campaign/generator.txt`. The server persists `campaignSpec`.
- Background stages use `generateCampaignStage` with `templates/campaign/stage_*.txt`.
- For initial adventures, `templates/dm/inject_initial.txt` is rendered from `campaignSpec` and injected into system messages.
- `skills/adventure_seed.txt` defines the adventure seed (structure, **story hook vs. paperwork** so openings are not “papers please” alone, **First opening** appendix: no PC proper name in prose before the first user message).
- `skills/character.txt` contains system-level character rules and a user-level template section (marked in the file). The server renders the user section as the user message.

## Editing

- Use Mustache-compatible variables (e.g. `{{campaignConcept}}`, `{{#factions}}...{{/factions}}`, `{{languageInstruction}}`).
- If you rename a file, update every `loadPrompt('…')` call site (e.g. `server/routes/gameSession.js`) and this doc.

## Prompt catalog (by path)

| Path | Purpose | Used by |
|------|---------|---------|
| `core/system.txt` | Core DM rules, JSON play envelope (no full combat mechanics — see `skills/combat.txt`) | `composeSystemMessages` (first system message) |
| `core/style.txt` | Global narrative style | `composeSystemMessages` |
| `rules/no_prefatory_guard.txt` | Forbids prefatory/meta-text | Campaign routes; consolidated stacks as loaded by code |
| `rules/json_output_guard.txt` | Strict JSON for campaign/stages | Campaign core, stages |
| `rules/json_output_guard_dm_play.txt` | DM `/generate`: envelope JSON; no bare `{}` | `/generate` |
| `rules/language_spanish.txt` / `rules/language_english.txt` | Language directives | Many routes |
| `rules/length_guard.txt` | Reply length / continuation | `composeSystemMessages` (late) |
| `rules/memory_summary.txt` | Session summary wrapper | When `sessionSummary` set |
| `rules/scene_grounding.txt` | Geography / beat discipline | Appended to consolidated `/generate` system text |
| `rules/intent_router_system.txt` | Intent classification | `intentRouter.js` |
| `templates/campaign/generator.txt` | Campaign metadata builder + Mustache user slice (`---` separated) | `/generate-campaign`, `/generate-campaign-core` |
| `templates/campaign/stage_factions.txt` | `factions` array | `generateCampaignStage('factions')` |
| `templates/campaign/stage_majorNPCs.txt` | `majorNPCs` array | `generateCampaignStage('majorNPCs')` |
| `templates/campaign/stage_keyLocations.txt` | `keyLocations` array | `generateCampaignStage('keyLocations')` |
| `templates/dm/inject_initial.txt` | Opening campaign injection | `/generate` initial |
| `templates/dm/inject_explore.txt` | Exploration injection | `/generate` exploration |
| `templates/dm/inject_combat.txt` | Combat/tense encounter injection | `/generate` combat/decision/investigation |
| `templates/dm/player_response_envelope.txt` | DM JSON envelope schema | `/generate` |
| `templates/dm/party_context.txt` | Multi-PC names / opening | Party block |
| `templates/dm/player_character_context.txt` | Per-player sheet + encounter state | `/generate` |
| `templates/dm/combat_entry_handoff.txt` | Combat entry handoff | `/generate` when entering combat |
| `skills/adventure_seed.txt` | Opening seed + first-opening naming rule | Initial `/generate` (full skill block) |
| `skills/combat.txt` | Combat mechanics | Combat mode |
| `skills/investigation.txt` | Investigation | Investigation mode |
| `skills/decision.txt` | Decision framing | Decision mode |
| `skills/character_generation_scope.txt` | Character route scope (not a play turn) | `/generate-character` |
| `skills/character.txt` | Character generation system + user section | `/generate-character` |

## Troubleshooting

- If a stage fails, `GameState.rawModelRequest` and `GameState.rawModelOutput` are populated for debugging.
- The server logs outgoing messages as `OUTGOING` / `DEBUG` entries — check terminal logs.
