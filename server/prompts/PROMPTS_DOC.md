# Dungeon Master Prompt Catalog

This document describes all server-side prompt templates, their purpose, canonical filenames, and where they are used. This file is the canonical prompt catalog and load order for the project.

The table below is the canonical prompt catalog and recommended load order for the project. Edit the prompt files under `server/prompts/` to change behavior; the server prefers server-side copies when loading prompts.

## Notes on usage
- All prompts live under `server/prompts/`. `promptManager.loadPrompt(filename)` prefers server copies first, then client copies.
- System messages are consolidated via `consolidateSystemMessages(...)` before sending to the model to preserve guard precedence.
- Campaign core is generated by `/generate-campaign-core` (template `campaign_generator_prompt.txt`). The server persists `campaignSpec`.
- Background stages are generated after the core via `generateCampaignStage` using the stage templates. These are persisted into `campaignSpec.<stage>`.
- For initial adventures, `dm_inject_initial.txt` is rendered from `campaignSpec` and injected into system messages so the adventure relates to the campaign.
- `skill_adventureSeed.txt` defines the required structure for the adventure seed (must start mundanely, then be drawn into campaign elements).
- `skill_character.txt` contains both the system-level character rules and a user-level template section (marked in the file). The server renders the user section as the user message.

## Editing prompts
- Edit `server/prompts/<file>` to change model instructions. Use Mustache-compatible variables (examples: `{{campaignConcept}}`, `{{#factions}}...{{/factions}}`, `{{languageInstruction}}`).
- If you rename a file, update `PROMPTS_MANIFEST.json` and any code references (or add an alias).

## Prompt catalog (table)
| Filename | Purpose | Injected / Used by | Key template variables |
|---|---|---:|---|
| systemCore.txt | Core system rules, role and high-level policies (no combat mechanics — see `skill_combat.txt`) | All requests (first system message) | (none) |
| styleStory.txt | Global narrative style & voice guidance | All requests | (none) |
| no_prefatory_guard.txt | Forbids prefatory/meta-text from the model | All requests (early) | (none) |
| json_output_guard.txt | Enforces strict JSON-only outputs when required | Campaign core, stage endpoints | (none) |
| language_spanish.txt | Strong language directive for Spanish sessions | Injected when language=Spanish | (none) |
| language_english.txt | Strong language directive for English sessions | Injected when language=English | (none) |
| length_guard.txt | Reply length and continuation guidance | All requests (late) | (none) |
| campaign_generator_prompt.txt | Template requesting the small campaign core | `/generate-campaign-core` (user message) | `{{sessionSummary}}`, `{{languageInstruction}}` |
| campaign_stage_factions.txt | Produce `factions` array for campaignSpec | `generateCampaignStage(stage='factions')` | `{{campaignConcept}}`, `{{languageInstruction}}` |
| campaign_stage_majorNPCs.txt | Produce `majorNPCs` array for campaignSpec | `generateCampaignStage(stage='majorNPCs')` | `{{campaignConcept}}`, `{{languageInstruction}}` |
| campaign_stage_keyLocations.txt | Produce `keyLocations` array for campaignSpec | `generateCampaignStage(stage='keyLocations')` | `{{campaignConcept}}`, `{{languageInstruction}}` |
| dm_inject_initial.txt | DM-only context for initial scenes (campaignSpec -> system message) | `/generate` when mode='initial' | `{{campaignConcept}}`, `{{factions}}`, `{{majorConflicts}}`, `{{majorNPCs}}`, `{{keyLocations}}`, `{{languageInstruction}}` |
| dm_inject_explore.txt | DM-only exploration guidance (factions) | `/generate` when mode='exploration' | `{{factions}}`, `{{languageInstruction}}` |
| dm_inject_combat.txt | DM-only combat/decision guidance (NPCs, conflicts) | `/generate` when mode='combat'/'decision' | `{{majorNPCs}}`, `{{majorConflicts}}`, `{{languageInstruction}}` |
| skill_adventureSeed.txt | Full adventure seed template (structure & mundane start rule) | Injected for initial scenes as system prompt | `{{languageInstruction}}` |
| skill_combat.txt | Combat mechanics: resource tracking (HP, spell slots, conditions), initiative/turn order, concise HP lists, and roll presentation conventions (single-line mechanical summaries + short narration) | `/generate` when mode='combat' | `{{languageInstruction}}` |
| skill_investigation.txt | Investigation guidance | `/generate` when mode='investigation' | `{{languageInstruction}}` |
| skill_decision.txt | Decision framing guidance | `/generate` when mode='decision' | `{{languageInstruction}}` |
| skill_character.txt | Character-generation system + embedded user prompt | `/generate-character` (system + user extracted) | `{{gameSetup}}`, `{{languageInstruction}}` |
| skill_character_examples_en.txt | Few-shot examples (English) for character style | Included for decision/initial as assistant few-shot | (none) |
| skill_character_examples_es.txt | Few-shot examples (Spanish) for character style | Included for decision/initial as assistant few-shot | (none) |

## Troubleshooting
- If a stage fails, `GameState.rawModelRequest` and `GameState.rawModelOutput` are populated for debugging.
- The server logs outgoing messages (console) as `OUTGOING` entries — check terminal logs.
