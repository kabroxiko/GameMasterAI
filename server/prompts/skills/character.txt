JSON structure (MANDATORY — follow before you output):
- **Shape:** Exactly one top-level key: `"playerCharacter"`, whose value is the full character object. No sibling keys at the root.
- **Closing braces (most common failure):** The root is `{ "playerCharacter": { ...inner... } }`. After the last field inside `playerCharacter`, you must emit one `}` to close `playerCharacter`, then one more `}` to close the root. **If the last field is itself an object** (`spell_slots`, `stats`, `coinage`, etc.), you must first close **that** inner object, **then** close `playerCharacter`, **then** close the root. Count braces: every `{` needs a matching `}`; nested objects require an extra `}` before the root closes. Stopping one `}` short of the full root is invalid JSON and will be rejected.
- **Self-check:** Before sending, verify every `{` has a matching `}` (ignore braces inside JSON strings). Mentally run `JSON.parse` on your exact output string; if it would throw, fix the structure until it would succeed.
